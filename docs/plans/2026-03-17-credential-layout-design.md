# 证件模块 - 生成与排版设计方案

## 概览
根据需求反馈，证件的生成和排版功能将在整个架构中分为两端来分别实现：
1. **DOOR (Web 管理端)**: 负责业务逻辑和需求定义——创建物理区域（如：嘉宾区、媒体区）、岗位角色、证件需求配置（如A区需要50张媒体证）、以及继承现有的扫码核销逻辑。
2. **TOOL (Electron 客户端)**: 视为重度的生产力工具。它将从 DOOR 端拉取需求数据，允许用户进行可视化的证件排版（类似于现有的 `BibLayoutView` 号码布排版），获取交互式的地图区域截图，并在本地执行高性能的照片级 PDF 批量导出。

## 组件架构

### 1. DOOR 端 (业务逻辑与定义)
- **数据模型**:
  - `CredentialZone` (证件区域): 定义物理上的区域（例如“主舞台”）。需要关联中心坐标 / 边界框。
  - `CredentialRole` (证件岗位): 定义岗位名称/职责。
  - `CredentialRequirement` (证件需求): 将某个岗位与某个区域关联，并配置分配名额。
  - `CredentialRecord` (证件记录): 生成出每一张实体证件的记录数据（包含唯一 ID、扫码 Token、若有姓名关联人员、当前状态）。
- **后台视图 (Views)**:
  - `CredentialZonePage`: 管理区域列表，以及在地图上大致圈定/记录初始坐标。
  - `CredentialRolePage`: 管理岗位类型。
  - `CredentialRequirementPage`: 生成所需“配额”（例如 50 张某区媒体证）。
  - `Scan Integration` (扫描集成): 扩展目前的扫码验证模块，增加对“证件类型”二维码的支持（类似目前号码布领物的逻辑）。

### 2. TOOL 端 (排版与批量生成工具)
- **数据模型 (模板与输出)**:
  - `CredentialTemplate` (证件模板): 本地存储或通过 API 与服务端同步。包含正反面的布局配置 (`frontLayoutJson`, `backLayoutJson`) 以及纸张尺寸。
- **视图组件 (Views)**:
  - `CredentialLayoutView.tsx` (现有组件): 负责类似号码布的排版引擎进行构建。
    - **升级目标**: 支持上传正反两面的底图（Background plates）。
    - 可拖拽编辑的字段类型：完善现有的文本（如姓名、编号、区域）、专属二维码 (QR Code) 字段，新增 **地图截图 (Map Viewport)** 字段。
    - **地图截图集成 (选项A)**:
      - 增加新字段类型：`mapImage`。
      - 当选择该字段时，可以点击一个配置按钮，弹出一个包含基于 `MapGl` 交互式地图组件的模态窗口。
      - 用户拖拽/缩放地图到指定区域 -> 工具直接将当前地图画布截取为高清图片 -> 嵌入至排版图层。
  - `CredentialGenerateView.tsx` / `CredentialExecution.tsx` (现有组件 / 批量生成引擎):
    - 从 DOOR 服务器轮询、拉取需要的 `CredentialRecord` 数据。
    - 为每个记录动态生成/获取相应的二维码。
    - 把正面、背面的布局一起塞入批量 PDF 生成管线渲染并导出。

## UI/UX 体验流 (聚焦于 TOOL 端)
1. **选择赛事与模板**: 打开 TOOL 客户端 -> 进入“证件排版”菜单 -> 选择当前赛事 -> 创建或读取模板。
2. **正反面排版界面**:
   - 左侧为工具和对象列表，右侧宽广区域为双面（或可切换标签）的画布。
3. **地图捕获流程**:
   - 设计师从左侧拖入一个“地图框”元素到背面设计中。
   - 点击该元素后，唤醒地图配置面板，渲染 2D 互动地图，设计师调整合适的视野缩放以完整包含该特定区域（例如安保 1 区）。
   - 确定后，系统捕获该定格的地图画面（甚至支持绘制好的多边形路径一起截图渲染），在排版画布中替代占位符显示。
4. **批量生成与导出**:
   - 在导出前可套用真实数据（从 DOOR 获取）预览最终生成的几张样例证件。
   - 利用 Worker 线程在本地文件系统高效并行渲染导出，最后生成可供印厂使用的高清 PDF 文档。

## 数据流向与新增 API
- `GET /api/v1/races/:raceId/credential-records`: Tool 工具向门控后台获取这一场赛事所有需要制作的证件列表。
- `POST /api/v1/races/:raceId/credential-records/sync`: Tool在生成实体证件和专属二维码后，将具体的 Token、编号匹配记录回传刷新到 DOOR 系统内，用来支持发证溯源扫码。
